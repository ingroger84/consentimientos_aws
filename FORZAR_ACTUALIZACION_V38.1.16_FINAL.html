✅ CORRECCIÓN DEFINITIVA

Modal de admisiones ahora funciona correctamente

🎉 Problema Resuelto

El problema NO era caché. Era un problema de timing en el código.

El modal se cerraba correctamente, pero la navegación ocurría tan rápido que React no tenía tiempo de procesar el cierre antes de cambiar de ruta.

🔧 Solución Implementada

  • Agregado setShowAdmissionModal(false) para cerrar el modal explícitamente
  • Agregado setTimeout(..., 100) para dar tiempo a React de procesar el cierre
  • La navegación ahora ocurre 100ms después de cerrar el modal
  • Esto permite que React actualice el DOM antes de cambiar de ruta
  • El modal desaparece correctamente y luego navega a la HC

📝 Código Corregido

// Cerrar el modal primero
setShowAdmissionModal(false);

toast.success('Admisión creada', 'Redirigiendo...');

// Pequeño delay para que React procese el cierre
setTimeout(() => {
  navigate(`/medical-records/${existingHC.id}?admissionId=${admission.id}`, { replace: true });
}, 100);

✅ Flujo Correcto Ahora

  • Usuario selecciona paciente con HC existente → Modal se abre
  • Usuario selecciona tipo de admisión y motivo
  • Usuario hace clic en "Crear Admisión"
  • Se crea la admisión en el backend
  • Se cierra el modal (setShowAdmissionModal(false))
  • Se espera 100ms para que React procese el cierre
  • Se navega a la HC con el admissionId
  • Se pueden agregar anamnesis, exámenes, diagnósticos, etc.
