📋 Problema Detectado

Los logins personalizados no se están mostrando debido a que tu navegador está usando una versión antigua en caché.

Síntoma: Todos los subdominios muestran "Sistema de Consentimientos" con logo genérico "S".

✅ Buenas noticias: El backend está funcionando correctamente y devuelve la configuración personalizada de cada tenant.

⚠️ Importante: Debes limpiar el caché del navegador para ver los cambios.

Pasos para Solucionar:

1

Limpiar Caché del Navegador

Chrome/Edge: Ctrl + Shift + Delete → Seleccionar "Todo el tiempo" → Marcar "Imágenes y archivos en caché" → Borrar datos

Firefox: Ctrl + Shift + Delete → Seleccionar "Todo" → Marcar "Caché" → Limpiar ahora

2

Forzar Recarga Sin Caché

Presiona Ctrl + Shift + R (Windows/Linux) o Cmd + Shift + R (Mac) en cada subdominio.

3

Verificar en Modo Incógnito

Abre una ventana de incógnito y accede a los subdominios. Si funciona aquí, confirma que el problema era el caché.

4

Verificar Logs de la Consola

Abre la consola del navegador (F12) y busca logs que empiecen con [ThemeContext] para ver qué está cargando.
