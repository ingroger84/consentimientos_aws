Sistema de Admisiones Múltiples Implementado

🎯 Resumen de Cambios Implementados

✨ Modal de Admisión Automático

Al abrir una HC existente sin admisión activa, el sistema muestra automáticamente el modal para crear una nueva admisión.

🔒 Validación de Admisión Activa

Los botones para agregar registros (anamnesis, exámenes, diagnósticos, evoluciones, consentimientos) se deshabilitan si no hay admisión activa.

⚠️ Alertas Visuales

Se muestran alertas amarillas en cada tab cuando no hay admisión activa, con botón para crear una nueva.

📋 Gestión de Admisiones

Tab "Admisiones" con funcionalidad completa: seleccionar admisión activa, cerrar, reabrir, y ver estadísticas.

🔄 Flujo Completo

Al seleccionar un cliente con HC existente en "Nueva Historia Clínica", se muestra el modal para crear una nueva admisión.

🧪 Pruebas a Realizar

⚠️ IMPORTANTE: Limpie la caché del navegador antes de realizar las pruebas (Ctrl+Shift+R o Cmd+Shift+R)
  • 1 Crear Nueva Historia Clínica
    ✓ Esperado: El botón "Nueva Historia Clínica" debe estar visible en la página de listado de HC.
  • 2 Seleccionar Cliente con HC Existente
    ✓ Esperado: Al buscar y seleccionar un cliente que ya tiene HC, debe mostrarse el modal de tipo de admisión automáticamente.
  • 3 Abrir HC Existente sin Admisión Activa
    ✓ Esperado: Al abrir una HC que no tiene admisiones activas, debe mostrarse automáticamente el modal para crear una nueva admisión.
  • 4 Intentar Agregar Registros sin Admisión
    ✓ Esperado: Los botones "Agregar Anamnesis", "Agregar Examen", etc. deben estar deshabilitados (gris) y mostrar tooltip al pasar el mouse.
  • 5 Ver Alertas en Tabs
    ✓ Esperado: En los tabs de Anamnesis, Exámenes, Diagnósticos y Evoluciones debe aparecer una alerta amarilla indicando que se requiere una admisión activa.
  • 6 Crear Nueva Admisión
    ✓ Esperado: Al hacer clic en "Nueva Admisión" o en el botón de la alerta, debe mostrarse el modal con los 10 tipos de admisión disponibles.
  • 7 Seleccionar Tipo de Admisión
    ✓ Esperado: Debe poder seleccionar un tipo (Primera Vez, Control, Urgencia, etc.) e ingresar el motivo de la admisión.
  • 8 Agregar Registros con Admisión Activa
    ✓ Esperado: Después de crear la admisión, los botones deben habilitarse y permitir agregar anamnesis, exámenes, diagnósticos, evoluciones y consentimientos.
  • 9 Ver Tab de Admisiones
    ✓ Esperado: El tab "Admisiones" debe mostrar todas las admisiones con estadísticas (cantidad de anamnesis, exámenes, diagnósticos, evoluciones, consentimientos).
  • 10 Seleccionar Admisión Activa
    ✓ Esperado: Debe poder expandir una admisión y hacer clic en "Seleccionar como Activa" para cambiar la admisión activa.
  • 11 Cerrar Admisión
    ✓ Esperado: Debe poder cerrar una admisión activa, lo que la bloqueará y no permitirá agregar más registros a ella.
  • 12 Reabrir Admisión
    ✓ Esperado: Debe poder reabrir una admisión cerrada, lo que la reactivará y permitirá agregar registros nuevamente.
  • 13 Múltiples Admisiones para el Mismo Paciente
    ✓ Esperado: Debe poder crear múltiples admisiones para el mismo paciente/HC, cada una con su propio tipo y motivo.
  • 14 Vinculación de Registros a Admisión
    ✓ Esperado: Todos los registros agregados (anamnesis, exámenes, etc.) deben aparecer en las estadísticas de la admisión activa.

🔄 Flujo Esperado Completo

Escenario 1: Nuevo Paciente
  1. Usuario hace clic en "Nueva Historia Clínica"
  2. Busca o crea un nuevo paciente
  3. Completa el formulario y crea la HC
  4. Sistema crea automáticamente la primera admisión
  5. Usuario puede agregar registros inmediatamente
Escenario 2: Paciente con HC Existente (desde "Nueva HC")
  1. Usuario hace clic en "Nueva Historia Clínica"
  2. Busca un paciente que ya tiene HC
  3. Sistema detecta HC existente y muestra modal de admisión
  4. Usuario selecciona tipo de admisión e ingresa motivo
  5. Sistema crea nueva admisión y redirige a la HC
  6. Usuario puede agregar registros a la nueva admisión
Escenario 3: Abrir HC Existente sin Admisión Activa
  1. Usuario abre una HC desde el listado
  2. Sistema detecta que no hay admisión activa
  3. Muestra automáticamente modal de admisión
  4. Usuario selecciona tipo e ingresa motivo
  5. Sistema crea admisión y habilita botones
  6. Usuario puede agregar registros
Escenario 4: Abrir HC con Admisión Activa
  1. Usuario abre una HC desde el listado
  2. Sistema detecta admisión activa existente
  3. Botones están habilitados automáticamente
  4. Usuario puede agregar registros inmediatamente
  5. Si necesita nueva admisión, hace clic en "Nueva Admisión"
