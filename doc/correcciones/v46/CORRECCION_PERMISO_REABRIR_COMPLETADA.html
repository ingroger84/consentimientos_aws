📋 Problema Reportado

El usuario Operador podía reabrir admisiones a pesar de no tener el permiso correspondiente en su rol.

Permisos del rol Operador en Historias Clínicas:

  • ✅ Ver historias clínicas (view_medical_records)
  • ✅ Crear historias clínicas (create_medical_records)
  • ✅ Vista previa de historias clínicas (preview_medical_records)
  • ✅ Enviar historias clínicas por email (send_email_medical_records)
  • ❌ NO tiene edit_medical_records
  • ❌ NO tiene reopen_medical_records

🔍 Causa Raíz

Problema 1: Backend - Permiso incorrecto

El endpoint de reabrir admisiones estaba usando el permiso EDIT_MEDICAL_RECORDS, que es muy amplio.

@Patch(':id/reopen') @RequirePermissions(PERMISSIONS.EDIT_MEDICAL_RECORDS) // ❌ Permiso muy amplio async reopen(@Param('id') id: string, @Request() req) { return this.admissionsService.reopen(id, req.user.tenantId); }

Problema 2: Frontend - Botón visible sin verificar permiso

El componente pasaba la función onReopenAdmission sin verificar si el usuario tenía el permiso, haciendo que el botón apareciera para todos.

<AdmissionsSection admissions={record.admissions || []} onReopenAdmission={handleReopenAdmission} // ❌ Siempre pasaba la función />

✅ Solución Implementada

Solución 1: Backend - Permiso específico

Cambié el endpoint para usar el permiso específico REOPEN_MEDICAL_RECORDS:

@Patch(':id/reopen') @RequirePermissions(PERMISSIONS.REOPEN_MEDICAL_RECORDS) // ✅ Permiso específico async reopen(@Param('id') id: string, @Request() req) { return this.admissionsService.reopen(id, req.user.tenantId); }

Archivo modificado: backend/src/medical-records/admissions.controller.ts

Solución 2: Frontend - Verificar permiso antes de mostrar botón

Modifiqué el componente para solo pasar la función si el usuario tiene el permiso:

const canReopenRecords = hasPermission('reopen_medical_records'); <AdmissionsSection admissions={record.admissions || []} onReopenAdmission={canReopenRecords ? handleReopenAdmission : undefined} // ✅ Solo si tiene permiso />

Archivo modificado: frontend/src/pages/ViewMedicalRecordPage.tsx (línea 530)

Cuando onReopenAdmission es undefined, el componente no muestra el botón "Reabrir Admisión".

✅ Conclusión

El problema se solucionó con dos cambios complementarios:

  1. Backend: Cambié el permiso requerido en el endpoint de reabrir admisiones a REOPEN_MEDICAL_RECORDS
  2. Frontend: Modifiqué el componente para solo mostrar el botón si el usuario tiene el permiso

Ahora solo los roles con el permiso reopen_medical_records pueden:

  • Ver el botón "Reabrir Admisión" en el frontend
  • Ejecutar la acción de reabrir en el backend

El rol Operador ya NO puede reabrir admisiones (ni ve el botón ni puede ejecutar la acción).
