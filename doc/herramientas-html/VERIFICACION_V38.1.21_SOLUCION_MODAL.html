Solución Modal Admisiones

19 de Febrero, 2026

🔍 Problema Identificado

Síntoma: Al crear una admisión desde el modal en CreateMedicalRecordPage, el modal no se cerraba y la página se quedaba bloqueada mostrando "Paciente con Historia Clínica Existente".

Confirmaciones:

  • ✅ La admisión SÍ se creaba exitosamente en el backend
  • ✅ Aparecía mensaje verde "Nueva admisión creada"
  • ❌ El sistema NO navegaba a la página de la HC
  • ❌ El modal permanecía visible y bloqueaba la interfaz
  • ❌ NO era problema de caché (probado en 3 computadores con diferentes ISP)

Causa Raíz:

// Código problemático (V38.1.20) window.location.href = `/medical-records/${existingHC.id}?admissionId=${admission.id}`; // Problema: window.location.href intenta navegar ANTES de que React // desmonte el modal del DOM. El modal tiene position:fixed y z-index:50, // por lo que permanece visible durante la navegación.

✨ Solución Implementada

❌ ANTES (V38.1.20)

// Navegación inmediata window.location.href = `/medical-records/${id}`; // Modal queda montado

✅ AHORA (V38.1.21)

// 1. Cerrar modal setShowAdmissionModal(false); // 2. Navegar con React Router setTimeout(() => { navigate(`/medical-records/${id}`); }, 100);

Cambios Técnicos:

  1. Reemplazado window.location.href por navigate()

    Usar el router de React permite que el framework maneje correctamente el ciclo de vida de los componentes.

  2. Cerrar modal ANTES de navegar

    Llamar a setShowAdmissionModal(false) desmonta el modal del DOM antes de la navegación.

  3. Delay de 100ms

    Dar tiempo a React para actualizar el DOM y desmontar el modal completamente.

  4. Mensaje de feedback

    Mostrar toast "Redirigiendo a la historia clínica..." para mejor UX.
